파일 생성

- Docker는 **애플리케이션을 `컨테이너화하여 개발, 배포, 실행할 수 있게 도와주는 플랫폼`**이다.
- 컨테이너는 애플리케이션과 그에 필요한 라이브러리, 의존성을 모두 포함하여, **`어디서나 동일한 환경에서 실행`**될 수 있는 패키지 형태라고 볼 수 있다.
- 이를 통해 운영체제의 환경 차이에 영향을 받지 않고 애플리케이션을 일관되게 실행할 수 있는 것이다.

## Docker를 사용해야 하는 이유

- **`환경 일관성 유지**:` 컨테이너 안에 필요한 모든 요소가 포함되어 있어, 다른 개발자 또는 서버 환경에서도 동일하게 애플리케이션이 작동한다.
- **`빠른 배포**:` Docker는 빠르게 애플리케이션을 배포할 수 있도록 하고, 빌드 및 배포 시간을 단축해 준다.
- **`효율적인 자원 활용**:` 컨테이너는 기존 가상 머신(VM)보다 가볍고 빠르며, 호스트 OS 자원을 효율적으로 사용한다.

## Docker의 주요 개념

- **`이미지(Image)**:` 애플리케이션 실행에 필요한 설정, 코드, 의존성을 포함한 패키지로, 컨테이너를 생성하는 기본 템플릿이다.
- **`컨테이너(Container)**:` 이미지를 기반으로 생성된 실행 가능한 인스턴스로, 독립된 환경에서 애플리케이션을 실행한다.
- **`Dockerfile**:` 이미지를 만들기 위한 설정 파일로, 필요한 의존성, 설치 패키지 등을 정의하여 커스텀 이미지를 생성한다.

## Docker 사용의 장/단점

**`장점**:`

- 어떤 환경에서도 동일하게 실행 가능하다.
- 컨테이너는 VM보다 가볍고 빠르며, 시스템 자원을 적게 사용한다.
- 컨테이너 이미지 버전 관리를 통해 변경 내역을 추적 가능하다.
- 복잡한 환경 설정 없이 애플리케이션 배포 가능하다.

**`단점**:`

- 특정 네트워크 설정을 위해 추가적인 작업 필요하다.
- Docker는 리눅스 커널을 기반으로 하므로, 윈도우에서 사용하는 경우 일부 제한사항이 존재한다.

## **Docker Compose란 무엇인가?**

- Docker Compose는 여러 개의 컨테이너를 한꺼번에 정의하고 실행할 수 있게 해주는 **도구**이다.
- Docker Compose는 단일 파일(`docker-compose.yml`)에 컨테이너 설정을 정의하여 **다중 컨테이너 애플리케이션을 손쉽게 관리**할 수 있다.
- 예를 들어, 웹 애플리케이션, 데이터베이스, 캐시 서버 등 여러 서비스가 함께 동작하는 경우, Docker Compose를 사용해 각 서비스를 컨테이너로 정의하고 동시에 관리할 수 있다.

## Docker Compose의 주요 개념

- **`서비스(Service)**:` 하나의 애플리케이션을 이루는 각 컨테이너를 의미하며, 예를 들어 `web`, `db`와 같은 서비스로 구분할 수 있다.
- **`네트워크(Network)**:` Compose에서 정의된 각 컨테이너 간의 통신을 담당하며, 각 컨테이너는 독립된 네트워크 내에서 서로 접근 가능하다.
- **`볼륨(Volume)**:` 데이터를 지속적으로 저장하기 위한 공간으로, 데이터베이스와 같은 지속적인 저장이 필요한 서비스에 활용한다. `(컨테이너 간에 공유할 데이터를 정의)`

## Docker Compose의 필요성

1. 여러 개의 컨테이너를 하나의 파일로 정의하고 관리할 수 있어 복잡한 설정을 간편하게 관리된다.
2. Compose 파일을 사용하여 쉽게 개발 및 테스트 환경을 재현할 수 있으며, 컨테이너 간 네트워크 설정을 자동화할 수 있다.
3. `docker-compose up` 명령으로 전체 애플리케이션 환경을 단일 명령어로 배포할 수 있다.

## 요약

Docker는 환경에 구애받지 않고 애플리케이션을 일관되게 실행할 수 있게 해주는 **`컨테이너 가상화 도구`**이며, Docker Compose는 이러한 Docker 컨테이너들을 **다중 구성**으로 관리하여 애플리케이션의 다양한 서비스 구성 요소들을 간편하게 배포할 수 있게 한다.
